## Core AgentBloc Concepts

**Agent**: A specialized AI assistant that performs one specific job in your workflow. Each agent has a clear role, like "Invoice Collector" or "Report Sender."

**Agent Team**: A group of agents that work together to automate your business process. Each agent handles one responsibility, and they coordinate through a defined topology.

**Artifact**: A configuration file generated by AgentBloc that tells your agents how to run. Examples include team.yaml (team roster), agent.yaml (individual instructions), and governance.yaml (safety rules).

**Blast-Radius**: A security score from 1 (safe, read-only) to 4 (can send messages externally). Agents scoring 3 or higher need your approval before they act.

**Contract**: A specification for what an agent does: its inputs, outputs, tools, schedule, and failure handling. Think of it as the agent's job description.

**Dry Run**: A test run where agents process real data but do not actually send emails, make payments, or modify external services. A mandatory safety step before going live.

**Evolution Loop**: A weekly automated check that watches for new capabilities or security issues affecting your agent team, then proposes changes for your approval.

**Gate**: An approval checkpoint between phases. You must explicitly confirm ("yes", "approved") before AgentBloc moves to the next phase. Gates prevent premature decisions.

**Governance**: The set of rules that control what your agents can and cannot do: budgets, permissions, rate limits, audit logging, and compliance requirements.

**Kill Switch**: An emergency stop button that halts all agent activity immediately. A simple file-based mechanism that agents check before every run.

**Phase**: One of the six stages in the AgentBloc process: Interview, Design, Integration Analysis, Confirmation + Dry Run, Deployment, and Evolution.

**Progressive Disclosure**: A design pattern where AgentBloc shows you only the information relevant to the current step, loading more detail as needed. Keeps conversations focused.

**State File**: A JSON file where agents save their progress, such as which invoices have been processed or which payments have been matched. Ensures agents never duplicate work.

**Subagent**: An agent that runs inside another agent's session with restricted tools and isolated context. Used for delegating specific subtasks safely.

**Topology**: The arrangement of how agents connect and communicate. The four types are Pipeline, Hierarchy, Mesh, and Swarm.

**Topology: Hierarchy**: A coordinator agent directs multiple worker agents and collects their results. Like a manager delegating tasks to a team.

**Topology: Mesh**: Agents collaborate as peers, each refining a shared output. Like a group editing the same document together.

**Topology: Pipeline**: Agents arranged in a chain where each one passes its output to the next. Like an assembly line.

**Topology: Swarm**: Agents explore independently in parallel, with results merged later. Used for research or data collection from many sources.

## Integration and Technical

**API**: A way for software programs to communicate with each other. Like a digital mailbox between services.

**Cron**: An automatic scheduler that runs your agents at specific times, like "every day at 10pm." Uses the standard Unix scheduling system.

**Decision Matrix**: A comparison table that ranks integration options for a service based on trust, setup effort, and capabilities. Helps pick the best connector.

**Fallback Chain**: A backup plan for each integration. If the preferred method fails (API is down), the agent automatically tries the next option (browser automation, then email scraping).

**MCP Server**: A connector that lets AI agents talk to external services like Google Sheets, Telegram, or Shopify. Stands for Model Context Protocol.

**OAuth**: A secure login method where you grant limited access to a service without sharing your password. The gold standard for agent credentials.

**Playwright**: A browser automation tool that lets agents interact with websites just like a human would, clicking buttons and reading pages. Used when no API exists.

**Trust Score**: A rating (HIGH, MEDIUM, LOW) assigned to each integration based on who maintains it, how actively it is updated, and how widely it is used.

**Webhook**: A notification that fires automatically when something happens in a service. Like a doorbell that rings when a new order arrives.

## Security and Compliance

**Audit Log**: A detailed, tamper-resistant record of everything your agents do: what they accessed, what they changed, and when. Required for compliance and debugging.

**Correlation ID**: A unique tracking code that links all actions from a single agent run together. Like a receipt number that ties all items in one purchase.

**Credential Hierarchy**: The principle of giving each agent the least powerful login it needs. Preference order: OAuth (safest) > scoped API key > admin token (riskiest).

**Data Classification**: The process of tagging every piece of data in your workflow as PII, PHI, financial, or public. Determines which security rules activate.

**DSAR (Data Subject Access Request)**: A formal request from a person to see, correct, or delete their personal data. Required under GDPR.

**GDPR**: The European Union's General Data Protection Regulation. Mandatory privacy rules for any business handling EU personal data.

**HIPAA**: A US regulation protecting health information. Activates automatically when your workflow handles patient or medical data.

**PCI**: Payment Card Industry Data Security Standard. Activates when your workflow handles credit card numbers or payment data.

**PII (Personally Identifiable Information)**: Any data that can identify a specific person: names, emails, phone numbers, addresses, government IDs.

**PHI (Protected Health Information)**: Health data linked to a specific person: diagnoses, prescriptions, medical records, health insurance IDs.

**Prompt Injection**: An attack where malicious instructions are hidden inside data an agent processes (emails, web pages), trying to trick the agent into doing something harmful.

**Rate Limiting**: A cap on how many actions an agent can perform in a given time period. Prevents runaway costs and accidental spam.

**Tenant Isolation**: Keeping one client's data completely separate from another's. Critical when a single AgentBloc deployment serves multiple businesses.

## Deployment and Operations

**ClaudeClaw / Job Definition**: A markdown file that tells Claude Code exactly what to do in one scheduled run. The agent reads this file as its mission briefing each time cron triggers it.

**Idempotency**: A guarantee that running the same task twice produces the same result without duplicating work. Like how pressing an elevator button multiple times only calls the elevator once.

**Notification Discipline**: The rule that agents stay silent unless something notable happens. No "everything is fine" messages flooding your phone.

**State Schema**: The defined structure of a state file, specifying what fields it contains and what values are valid. Ensures agents read and write state consistently.

**Telegram Thread**: A dedicated conversation thread in Telegram where a specific agent or domain sends its notifications. Keeps different agent reports organized.
