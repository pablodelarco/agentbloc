## When This Applies

Briefing-agent wake.md section 5 (Execute) calls `bash .agentbloc/runtime/activity-feed-merge.sh <YYYY-MM-DD>` to generate the day's feed before dispatching the briefing summary message. Loaded by deploy-engine when emitting briefing-agent SKILL.md.

## Path + Generation

Output: `.claude/agents/logs/<YYYY-MM-DD>/activity-feed.jsonl` (sibling to per-agent files + approvals.jsonl + escalations.jsonl).

Generation: merger globs `.claude/agents/logs/<YYYY-MM-DD>/*.jsonl` (excluding `activity-feed.jsonl` itself), parses each line, sorts ascending by `timestamp`, emits to the output file. RFC 8785 JCS canonicalization preserved per line.

## Byte-Identity Rule

Feed entries are BYTE-IDENTICAL to source per-agent log entries. No enrichment, no transformation, no field additions. The merge is the only operation. This means:
- The feed file is reconstructible at any time from the per-agent files.
- SHA256 fingerprints of feed lines match the originals.
- Phase 16 golden-file harnesses can compare line-by-line.

## Idempotency

Safe to re-run `activity-feed-merge.sh <DATE>` for the same date. Output is deterministic given input set. The merger overwrites prior feed file with the same content (since byte-identity holds). Idempotency is critical because briefing-agent may re-run on retry (e.g., Telegram dispatch failed and retry fires the whole wake).

## activity-feed-merge.sh Shell Contract

**Args:** `<YYYY-MM-DD>` (single positional)
**Behavior:** glob -> parse -> sort -> write
**Exit codes:** 0 = success; 1 = read failure (no per-agent files for date); 2 = write failure (disk full / permission)
**Side effects:** writes (or rewrites) `.claude/agents/logs/<YYYY-MM-DD>/activity-feed.jsonl`. NO state.json mutations; NO Telegram calls; NO log writes about itself.

Reference implementation:

```bash
#!/usr/bin/env bash
DATE="$1"; LOGS_DIR=".claude/agents/logs/${DATE}"
[ -d "$LOGS_DIR" ] || exit 1
FILES=$(ls "${LOGS_DIR}"/*.jsonl 2>/dev/null | grep -v activity-feed.jsonl)
[ -n "$FILES" ] || exit 1
cat $FILES | jq -c '.' | jq -s 'sort_by(.timestamp) | .[]' -c > "${LOGS_DIR}/activity-feed.jsonl.tmp" || exit 2
mv "${LOGS_DIR}/activity-feed.jsonl.tmp" "${LOGS_DIR}/activity-feed.jsonl" || exit 2
exit 0
```
