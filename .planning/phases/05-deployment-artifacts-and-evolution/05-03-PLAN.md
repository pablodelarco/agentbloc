3. **When This Applies:** Referenced during Phase 5 (telegram.yaml generation) and Phase 2 (reporting design). Not a standalone conversational phase
4. **Thread-Per-Domain Convention (D-10):** Each logical domain (e.g., "Invoices", "Payments", "Errors", "Approvals") gets its own Telegram forum topic (thread) within the team's supergroup. Messages are routed by domain using the Telegram Bot API `message_thread_id` parameter. Explain: supergroup must have "Topics" (forum mode) enabled. Include the Arco Rooms thread mapping example (invoices, payments, operations, approvals)
5. **Notification Tiers (D-11):** Three tiers with distinct formatting:
   - `info`: Plain text, routine updates. Example: "3 new invoices collected from Xero"
   - `action_required`: Bold header prefix (**ACTION REQUIRED:**). Example: "**ACTION REQUIRED:** 1 low-confidence match needs review"
   - `error`: Alert emoji prefix. Example: "ALERT: Invoice Collector failed to reach Endesa portal"
   Silence-by-default: no "everything is fine" messages. Only notable events trigger notifications
6. **Approval-by-Reply (D-12):** For Level 3-4 agents (blast-radius write-unrestricted or send-external, per blast-radius.md). When an agent needs human approval, it sends a structured approval request to the "approvals" thread. Request includes: agent name, action summary, preview of what will be sent/modified, approve/reject instructions. User replies to approve or reject. Timeout configurable in governance.yaml (approvals.default_timeout_minutes). On timeout: do nothing (safe default). On approval: proceed with action. On rejection: log reason, skip action
7. **Voice Message Support (D-13):** Document as a Telegram-native feature. Users can reply with voice messages for approvals or feedback. Claude processes the voice transcription (Telegram provides text transcription). Not a custom feature; leverages Telegram's built-in voice-to-text
8. **Bot Setup Requirements:** Bot must be admin of the supergroup with "Manage Topics" permission. Bot token stored in .env (never in telegram.yaml). Chat ID is the supergroup ID (negative number). Thread IDs obtained after creating forum topics in the supergroup
9. **Quick Reference:** Table of notification tiers with format and when to use

**Critical constraints:**
- Concise: target 80-120 lines. Pattern library, not tutorial
- Thread-per-domain, not thread-per-agent (per D-10)
- Silence-by-default is mandatory (per D-11)
- Approval timeout defaults to "do nothing" not "auto-approve"
- No deferred features (no multi-language notifications)
