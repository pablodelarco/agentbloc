4. **Phase 3: Integration Findings**: Per-agent integration decisions in decision matrix format (recommended + fallback). Trust scores for key dependencies. 5. **Phase 4: Confirmed Agent Cards**: Per-agent summary card showing actions, integrations chosen, blast-radius score, schedule, failure handling. Include a brief dry run note: "Dry run processed 5 invoices and 12 transactions. 4 matched automatically, 1 flagged for review. No side effects executed."
6. **Phase 5: Deployment Artifacts**: Show a directory tree of .agentbloc/ output. Include one short YAML excerpt (team.yaml with topology and agent refs, 8-10 lines max). Do NOT dump full YAML files.
7. **Phase 6: Evolution Notes**: What the weekly scan checks (new MCP servers for banking, PSD2 regulation updates, Playwright selectors for legacy portals). One example patch proposal.
