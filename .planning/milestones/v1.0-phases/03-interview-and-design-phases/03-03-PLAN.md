  | CrewAI Concept | AgentBloc Equivalent | Notes |

- When to apply: Always. The role-based decomposition pattern is the default agent identification method in AgentBloc.

**LangGraph Patterns for AgentBloc section:**
- Key borrowed concept: Stateful topology patterns. LangGraph models workflows as typed state graphs with nodes, edges, and conditional branching.
- Topology mapping table (from research, verified):

  | LangGraph Pattern | AgentBloc Topology | When to Use | Agent Count |
  |-------------------|-------------------|-------------|-------------|
  | Sequential Pipeline | Pipeline | Fixed sequential stages, clear handoff points | 1-3 |
  | Supervisor | Hierarchy | Centralized coordination, clear task boundaries | 3-5+ |
  | Swarm | Swarm | Unknown optimal paths, large-scale parallel collection | 5+ |
  | Mesh (peer-to-peer) | Mesh | Iterative refinement, shared artifacts, tight feedback loops | 3-8 |

- Topology selection decision tree (from research, verified):
  ```
  1. Are subtasks independent with no inter-agent communication? -> Pipeline
  2. Fixed, predictable sequence with clear boundaries? -> Pipeline
  3. 3-8 agents iterating on a shared artifact? -> Mesh
  4. Large problem space, unknown optimal path? -> Swarm
  5. 5+ agents across multiple domains? -> Hierarchy
  6. Default (when unclear): Pipeline (simplest, most predictable)
  ```

- Practical sizing guidance:
  - 1-3 agents: Default to Pipeline. Hierarchy adds overhead without enough routing complexity.
  - 3-5 agents: Consider Hierarchy (one coordinator + workers). Coordinator provides global view.
  - 5+ agents: Evaluate Hierarchy or Hybrid. Planning tier's cost justified when sub-teams can execute in parallel.

- State checkpoint concept (borrowed): AgentBloc uses JSON state files (.agentbloc/state/) for checkpointing, similar to LangGraph's persistent state. Each agent reads/writes state files, enabling crash recovery and idempotent re-runs.

- When to apply: During topology selection (Step 2). The decision tree above is the primary selection tool.

**n8n Patterns for AgentBloc section:**
- Key borrowed concept: Visual DAG mental model. n8n represents workflows as visual directed acyclic graphs mixing deterministic steps and AI nodes.
- What AgentBloc borrows: The explanatory mental model, not the runtime. When presenting the topology diagram to non-technical users, explain it as: "Each agent is a step in your workflow. Arrows show what information passes between them. Some steps are simple data collection (no AI needed), others use AI for matching or analysis."
- Mix of deterministic + AI steps: Not every agent needs an LLM. Some agents perform deterministic operations (file reads, API calls with fixed logic). Others need AI for interpretation, matching, or generation. This maps to model routing: Haiku for simple checks, Sonnet for standard processing, Opus for complex reasoning.
- When to apply: When explaining the design to non-technical users (Level: non-technical or technical-basics). For developer-level users, the topology diagram and contract cards are self-explanatory.
